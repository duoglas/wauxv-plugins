# GroupAdmin

群管理插件：踢人/黑名单/警告/潜水清理，按群管理员 + 全局群主。

> v1.16：**黑名单安全化**：拉黑危险，命令不能太简单。新增显式命令 `添加黑名单 @xx [@yy...]`（加黑+踢，可 @ 多人）、`去除黑名单 @xx [@yy...]`、`去除黑名单【N】`（按 `黑名单` 列表 1-based 序号移除，容错 `【】`/`[]` 全半角 + 前导 0）。`黑名单` 查询列表每项前加唯一序号 `【01】`。旧的过简单入口（`@xx 拉黑`/`@xx 黑名单`/`addsb`/`delbk`）保留识别但改为引导提示，不再直接拉黑/解黑。仅改黑名单增删入口，踢/警告/保护/管理员/潜水/SQLite 落盘等一律未动。
> v1.15：**性能**：潜水采集（`lsg_`/`fsg_`）落盘从 `config.prop` 整群 CSV 改为自有 **SQLite**（`groupadmin.db`），消息热路径只写内存增量（O(1)）+ 后台批量落盘，彻底消除大群高频消息时的微信收发卡顿；首次加载自动把旧 CSV 数据迁移进 DB。**警告增强**：① 除 `@TA` 外，**引用对方消息回复动作词**也能触发全部管理动作（踢/拉黑/警告/保护/管理员…，目标 = 被引用者）；② 警告自动踢上限默认 3 → **10**，且**每群可经设置 Dialog 或 `警告上限 N` 命令自定义**；③ 满上限自动踢出**成功后清零计数**（重新进群从 0 重新累计；机器人无权限未踢成则保留计数）。
> v1.14.0：**管理员权限按群拆分**：`@TA 管理员` 只写入当前群 `admins_<groupId>`；GUI 新增「本群管理员」和「全局群主」管理；移除微信管理员登记，微信原生管理员直接用本群管理员体系管理；保留微信原生群主保护。
> v1.13：**微信原生群主保护**：踢人、拉黑、警告、潜水清理前优先保护原生群主；尝试自动探测 WAuxiliary 群主字段，并支持 `@TA 微信群主` 手动登记 fallback。
> v1.12：**白名单**（本群 + 全局，免警告 + 免进群黑名单检测）；**全局黑名单**（任何启用群进入即踢）；**警告详情**（`@TA 警告详情` 查单人）；**清除警告名单**（一键全清，仅群主）；**严格模式**（开启后命令仅群主可用）；`doWarn` 记录最近警告时间 + 操作人；GUI 主 Dialog 加白名单按钮。
> v1.11：bot 在任意启用了群管的群里输入 `群管设置`（或 `群管配置` / `#群管设置`），通过 `onClickSendBtn` 拦截，弹出**本群图形化配置 Dialog** —— 启用开关、状态摘要、黑/保护/警告列表逐项操作（解黑/取消保护/清零）、潜水基线重建、bot 权限缓存重置、默认潜水天数设置。命令文本不会发出群里。

## 命令（群内发送）

### 群级开关（仅群主）
| 命令 | 用途 |
|---|---|
| `开启群管` / `启用群管` | 本群启用 GroupAdmin（默认关闭） |
| `关闭群管` / `禁用群管` | 本群停用（黑名单等数据保留） |
| `群管状态` | 查当前群启用状态 |

### 移出群聊 / 黑名单（管理员+）
| 命令 | 用途 |
|---|---|
| `@xxx 踢` / `@xxx 请` | 踢出群 |
| `添加黑名单 @xx [@yy...]` | 加黑名单 + 踢（再入群自动踢），可同时 @ 多人 |
| `去除黑名单 @xx [@yy...]` | 从本群黑名单移除，可同时 @ 多人 |
| `去除黑名单【N】[【M】...]` | 按 `黑名单` 列表序号移除（如 `去除黑名单【01】`，容错 `【1】`/`[01]`） |

> **v1.16 黑名单安全化**：拉黑是危险动作，入口从过于简单的方式（`@xx 拉黑`/`@xx 黑名单`/`addsb`/`delbk`）改为显式命令防误触。旧方式保留识别但只回引导提示，不再直接拉黑/解黑。`黑名单` 查询列表每项前带唯一序号 `【01】`/`【02】`…（序号 = 当前列表 1-based 位置），配合 `去除黑名单【N】` 使用。

### 白名单（v1.12，管理员+）
| 命令 | 用途 |
|---|---|
| `@TA 白名单` | 加入本群白名单 |
| `@TA 移除白名单` | 移出本群白名单 |
| `addwt wxid_xxx` | 按 wxid 加白名单 |
| `delwt wxid_xxx` | 按 wxid 移白名单 |
| `白名单` | 查本群白名单 |

白名单成员：**免警告 + 免进群黑名单检测**。如需拉黑必须先移除白名单。全局白名单（`addgw / delgw`）覆盖所有启用了群管的群。

### 全局配置（v1.12，管理员+）
| 命令 | 用途 |
|---|---|
| `addgb wxid_xxx` | 加入全局黑名单 |
| `delgb wxid_xxx` | 移出全局黑名单 |
| `全局黑名单` | 查看全局黑名单 |
| `addgw wxid_xxx` | 加入全局白名单 |
| `delgw wxid_xxx` | 移出全局白名单 |
| `全局白名单` | 查看全局白名单 |

**全局黑名单**：任何已启用群管的群进入即自动踢；**全局白名单**：任何启用群免警告 + 免进群检测。判断逻辑是「全局 OR 本群」合并视图。

### 警告增强（v1.12，管理员+）
| 命令 | 用途 |
|---|---|
| `@TA 警告详情` | 查单人警告详情（当前次数 + 最近时间 + 操作人） |
| `清除警告名单` | 一键清空本群所有警告（仅群主） |

### 严格模式（v1.12，仅群主）
| 命令 | 用途 |
|---|---|
| `严格模式 开` | 收紧所有命令到仅群主可用 |
| `严格模式 关` | 恢复默认（管理员+ 可用） |
| `严格模式 状态` | 查当前状态 |

全局生效（非按群），开启后即使本群有子管理员也无法用命令，等价对方插件的"群主专用"模式。

### 潜水清理（管理员+，v1.7 新增）
| 命令 | 用途 |
|---|---|
| `#潜水 7` | 列出过去 7 天没发言的人，编号 + 名片 |
| `#踢潜水 1,3,5-8` | 按编号批量踢（支持区间） |
| `@xxx 保护` | 加入豁免名单（潜水扫描不出现） |
| `@xxx 取消保护` | 取消保护 |
| `保护名单` | 查当前群保护名单 |

**潜水追踪机制：**
- 每条群消息记录发言时间 + 第一次见到的时间（first_seen）
- **开启群管时**拉取群成员列表，把所有人 first_seen 设为当时（建立基线）
- 插件加载时，对已启用但 fsg 为空的群延迟 5 秒兜底建立基线（防止漏掉历史启用的群）
- 新成员入群即记 first_seen，自动获得 N 天豁免期
- bot 自己、GroupAdmin 管理员/群主、保护名单成员永远不会出现在潜水名单
- 待踢编号缓存 30 分钟，过期需重发 `#潜水 N`

### 警告系统（管理员+）
| 命令 | 用途 |
|---|---|
| `@xxx 警告` | 警告 +1（累计达本群上限自动踢，默认 **10**，踢成功后计数清零） |
| `@xxx 警告-1` / `@xxx 警告减1` | 警告 -1 |
| `@xxx 清零` | 警告归零 |
| `警告上限 N` | 设本群警告自动踢上限（1~99，默认 10；也可在 `群管设置` Dialog 里改） |

> **引用回复同样触发动作**：以上所有 `@xxx 动作` 都可改用「**引用对方的一条消息 + 回复动作词**」触发，目标 = 被引用者。例如引用某人消息回复 `警告` / `踢`。权限与保护校验和 `@` 方式完全一致。（注：`拉黑`/`黑名单` 引用动作 v1.16 起只回引导提示，拉黑请用 `添加黑名单 @对方`。）

### 管理员管理（仅群主）
| 命令 | 用途 |
|---|---|
| `@xxx 管理员` | 提为本群管理员 |
| `@xxx 移除管理` | 移除本群管理员 |
| `@xxx 群主` | 提为全局群主 |
| `@xxx 移除群主` | 移除全局群主 |

### 微信原生群主保护（v1.13，仅群主）
| 命令 | 用途 |
|---|---|
| `@TA 微信群主` | 手动登记为微信原生群主，禁止任何踢/拉黑/警告/潜水清理 |
| `@TA 移除微信群主` | 取消微信群主保护登记 |

v1.14.0 保留微信原生群主自动探测和手动登记 fallback。微信原生管理员不再单独登记，统一使用「本群管理员」体系管理。

### bot 权限自检（管理员+，v1.7 新增）
| 命令 | 用途 |
|---|---|
| `#群权限自检` | 清除本群权限缓存，下次踢人时重新检测 |

**机制说明：** 微信原生没有暴露「bot 是不是群管理员」的 API。GroupAdmin 用 **lazy learning**：
- 第一次踢人时如果失败且错误信息含「权限/admin/permission」关键字 → 标记本群权限 = `none`
- 后续所有踢人命令在执行前先看缓存，是 `none` 直接拒绝并提示「请群主把 bot 设为群管」
- 成功踢一次 → 标记 `admin`
- 群主把 bot 提升后，发 `#群权限自检` 清缓存即可

### 查询
| 命令 | 用途 | 权限 |
|---|---|---|
| `群管帮助` / `帮助 群管` | 显示完整命令列表 | 管理员+ |
| `黑名单` | 列本群黑名单（每项带序号【01】，配合 `去除黑名单【N】`） | 管理员+ |
| `管理员` | 列管理员 + 群主 | 管理员+ |
| `警告名单` | 列警告计数 | 管理员+ |
| `保护名单` | 列豁免名单 | 管理员+ |

> v1.9 起所有查询/控制命令都需要管理员+，普通成员发命令插件静默忽略（不响应不报错，避免被群成员探测群管机制）。

## 权限层级

| 层级 | 来源 | 权限 |
|---|---|---|
| 全局群主（owner） | bot 自己 + `super_owners` config | 全权 + 管理本群管理员/全局群主 |
| 本群管理员（admin） | `admins_<groupId>` config | 仅在本群踢/拉黑/警告 + 管理普通人 |
| 微信原生群主 | 自动探测 + `nat_owner_<groupId>` | 受保护，不能被踢/拉黑/警告/潜水清理 |
| 普通用户 | 其他 | 只能查询 |

- bot 自己永远是全局群主（即使账号也是普通群员）
- 不能踢自己，不能踢同级别或更高的人
- 本群管理员只在当前群生效，不会串到其它群
- 微信原生管理员不再单独登记，直接按本群管理员管理

## 数据存储

所有数据存在插件目录的 `config.prop`：

| key | 内容 |
|---|---|
| `enabled_groups` | CSV: 启用 GroupAdmin 的群 ID |
| `super_owners` | CSV: 全局群主名单 |
| `admins_<groupId>` | CSV: 本群管理员名单 |
| `super_admins` | 旧版全局管理员 key，v1.14.0 起不再参与授权 |
| `bl_<groupId>` | CSV: 本群黑名单 wxid |
| `warn_<groupId>` | CSV: 本群警告记录 |
| `protected_<groupId>` | CSV: 本群保护名单 |
| `nat_owner_<groupId>` | CSV: 手动登记的微信原生群主保护名单 |
| `lsg_<groupId>` | `wxid|ts,...` 本群每人最后发言时间 |
| `fsg_<groupId>` | `wxid|ts,...` 本群每人首次记录时间 |
| `pending_kick_<groupId>` | CSV: 上次 `#潜水` 命令的待踢列表 |
| `bp_<groupId>` | bot 权限缓存: `admin`/`none`/`unknown` |
