新增一个**靠某配置键当闸门**的功能时,如果那个键**只在状态切换的那一刻写**(如开关 OFF→ON 写 `since` 时间戳),那么**升级时已经处于目标状态的用户(开关早就开着)从未经历切换 → 键没值 → 闸门把功能整个挡掉 → 静默失效**。表现:用户说"没生效",日志显示安全网在跳过(如 `no since anchor`)。

**Why:** 真实踩过(RP v1.11.4 伸手党):用户 v1.11.2 已 `伸手党 开`,v1.11.3 才加「开关打开时刻 `since`」闸门(只在 OFF→ON 写),升级带"已开"状态 → `since=0` → `since<=0` 安全网整包跳过 → 伸手党不发警告。代码全对,纯升级状态缺口。

**How to apply:**
1. **设计期**:任何"切换时写、读时当闸门"的键,先想"老用户升级时这个键会不会是空的"。会空就要么迁移时回填、要么读侧自愈。
2. **修法(读侧 lazy-init 自愈)**:闸门判定时若发现"功能开着但键为空",**当场补写键=now、本轮仍按保守分支跳过**(别因刚补就立即执行破坏性动作),下一次自然生效。免用户手动重切。比"让用户关→开重切"健壮——以后每次升级都自动兜。
3. **排查**:用户报"新功能没生效"先 grep 日志看是不是安全网在跳过 + 看那个闸门键有没有值,别一上来怀疑核心逻辑。
4. 顺带:写键/读键务必同一归一化(本次 Santa 抓到写裸 groupId、读 normGroupId 的潜在错配)。
